# GuruChakra Product Overview

A business-oriented description of the GuruChakra tuition platform: what it does, who uses it, and how it creates value.

---

## What Is GuruChakra?

GuruChakra is an online tuition marketplace that connects parents with qualified teachers for their children. Parents browse teachers by board, class, and subject; enroll students in batches; and manage classes. Teachers create batches, conduct live classes, and earn through the platform. Students attend classes, take AI-generated exams, and use AI-powered study materials.

The platform differentiates through **AI integration**: exam generation, evaluation, classroom monitoring, doubt answering, and study material generation—all powered by AI to reduce teacher workload and improve learning outcomes.

---

## User Roles

| Role | Who | Primary Use |
|------|-----|-------------|
| **Parent** | Guardian of one or more students | Find teachers, enroll children, manage classes, track performance |
| **Teacher** | Tutors who teach on the platform | Create batches, conduct classes, receive payments |
| **Student** | Child enrolled in tuition | Attend classes, take exams, use study materials |
| **Admin** | Platform operator | Manage users, teachers, enrollments, masters, schedules |

A single user account can have one role. Parents and students are linked; a parent manages their children’s enrollments.

## Core Capabilities by Role

### Parent

- **Registration:** Multi-step signup with OTP, profile, and children details
- **Marketplace:** Browse teachers by board, class, subject; filter and sort
- **Teacher profile:** View batches, fees, qualifications, reviews
- **Wishlist:** Save teachers for later
- **Checkout:** Select batch, slots, duration (3/6/12 months), complete payment
- **My kids:** Add, edit, remove children; view performance
- **My classes:** View upcoming classes, reschedule requests
- **Reviews:** Rate teachers after enrollment
- **Payments:** View payment history

### Teacher

- **Registration:** Multi-step signup with OTP, profile, qualification exam (AI-generated), document verification
- **Batches:** Create batches with slots, min/max students, fee, subject, board, class
- **Classes:** Start/end classes, view schedule
- **Students:** View enrolled students
- **Exams:** View student exam attempts and scores
- **Payments:** View earnings and payment history
- **Profile:** Update bio, documents, bank details

### Student

- **Dashboard:** Overview of courses and upcoming classes
- **Courses:** View enrolled subjects and batches
- **Classes:** Join live classes, start/end sessions
- **Exams:** Take quick tests, class tests, preparatory exams (AI-generated); view scores and feedback
- **Study materials:** Browse topics, generate AI study content, ask doubts

### Admin

- **Dashboard:** Platform metrics
- **Teachers:** List, approve/reject, view details, manage marketplace order
- **Parents / Students:** List, view, update
- **Enrollments:** List, manage status
- **Classes:** View class sessions
- **Masters:** Board, class, subject; mappings (board–class–subject)
- **Topics:** CRUD for study topics
- **AI data:** View AI-generated content
- **Drafts:** Manage draft content
- **Schedules:** Trigger schedule generation
- **Teacher payments:** Process payments

## Key Features

### Marketplace

Teachers list batches with board, class, subject, fee, and slots. Parents filter by these criteria. Only teachers with status `qualified` appear. Admin can set `marketplaceOrder` for relevance sorting.

### Enrollment and Payment

1. Parent selects batch and student
2. Chooses duration (3, 6, or 12 months) and slots
3. Checkout creates `PendingEnrollment`
4. Payment is completed via external provider
5. On success, `PendingEnrollment` becomes `Enrollment`
6. Schedules are generated for the enrollment period

### Class Scheduling

Schedules are generated by a cron job (or manual trigger). For each enrollment and slot, `ClassSession` records are created. Parents and teachers can request reschedules with proposed slots; the other party confirms or rejects.

### AI Features

| Feature | Description |
|---------|-------------|
| Qualification exam | AI generates MCQ exam for teacher registration |
| Student exam | AI generates questions by topic, type (quick/class/preparatory) |
| Exam evaluation | AI scores and provides feedback |
| Study material | AI generates content for topic + board + class |
| Doubt answering | AI answers questions in context |
| Classroom monitoring | AI analyzes video frames for attendance/behavior |
| Exam proctoring | AI monitors exam sessions for integrity |
| Document verification | AI verifies ID/photo documents |

## Business Model (Conceptual)

- **Parents** pay per enrollment (fee × duration, with optional discounts)
- **Teachers** receive a share of fees (platform fee structure is configurable)
- **Platform** earns from the margin between parent payment and teacher payout

---

## Supported Configurations

- **Boards:** CBSE, ICSE, State boards (configurable via masters)
- **Classes:** 1–12 (configurable)
- **Subjects:** Per board–class mapping (e.g. Math, Science, English)
- **Batch size:** 1–3 students per batch
- **Duration:** 3, 6, or 12 months
- **Discounts:** 0%, 5%, or 10% (configurable per enrollment)
